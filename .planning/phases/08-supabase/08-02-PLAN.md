<objective>
Wire Supabase into wikidata.ts so LLM-validated results are persisted globally and reused as a shared cache, eliminating redundant LLM API calls across users.

Purpose: Every LLM call costs money and time. Once one user validates "Hermione Granger" as fictional-women, the next user gets an instant DB hit instead of another LLM round-trip. Fire-and-forget write means zero latency impact on the existing LLM path.
Output: wikidata.ts modified at two points — DB read before LLM call, DB write after LLM success. Both modifications are null-safe and do not alter the existing return paths.
</objective>
