<feature>
  <name>Wikidata Search & Filter</name>
  <files>[src/services/wikidata.ts, src/services/wikidata.test.ts]</files>
  <behavior>
    The service must fetch search results from Wikidata `wbsearchentities` and then verify each result's properties via the `wbgetclaims` or SPARQL endpoint.
    - Input: string (name)
    - Output: Promise<{ id, label, description } | null> (or list if multiple matches)
    - Filter: Instance of (P31) = Human (Q5) AND Gender (P21) = Female (Q6581072).
    Cases:
    - "Billie Eilish" -> returns entity
    - "Angela Merkel" -> returns entity
    - "Donald Trump" -> returns null (fails gender filter)
    - "Statue of Liberty" -> returns null (fails human filter)
  </behavior>
  <implementation>
    Use `axios` for API calls.
    Implement `searchWomen(name: string)`:
    1. Call `wbsearchentities` to get candidates.
    2. For each candidate (up to first few), call `wbgetclaims` to check P31 and P21.
    3. Return the first matching female human or null.
  </implementation>
</feature>
