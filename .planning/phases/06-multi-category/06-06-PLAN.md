The uncommitted changes in both files are correct. The only code fix needed is in `src/services/wikidata.ts` around the `isHuman && isFemale` guard (currently lines 155-161).

**Current logic (broken for fictional categories):**
```typescript
const isHuman = claims.P31?.some((c: any) => validInstances.includes(c.mainsnak.datavalue?.value.id));
const isFemale = claims.P21?.some((c: any) => c.mainsnak.datavalue?.value.id === category.wikidataGender);

if (isHuman && isFemale) {
```

**Problem:** Many fictional characters on Wikidata have correct P31 (instanceOf = Q15632617 fictional human, or Q4663903 fictional video game character) but lack P21 entirely — so `isFemale` evaluates to `undefined` (falsy) and the character is rejected.

**Fix:** Make the gender filter conditional on whether the category declares `wikidataGender`. Replace those three lines with:

```typescript
const isHuman = claims.P31?.some((c: any) => validInstances.includes(c.mainsnak.datavalue?.value.id));

// Gender filter: only apply when category declares wikidataGender.
// Fictional categories omit wikidataGender because many characters lack P21 on Wikidata.
const passesGender = !category.wikidataGender ||
  claims.P21?.some((c: any) => c.mainsnak.datavalue?.value.id === category.wikidataGender);

if (isHuman && passesGender) {
```

This change means:
- Women / Men categories: `wikidataGender` is set → gender check still enforced (no behavioral change)
- Fictional Women / Fictional Men: `wikidataGender` is set → gender check still enforced for the ones that DO have P21, but that's still wrong for many fictional characters

Wait — fictional categories in the current `categories.ts` DO set `wikidataGender` (Q6581072 for fictional-women, Q6581097 for fictional-men). The real fix is to remove `wikidataGender` from the fictional category configs so the filter is skipped entirely, relying on the specific `wikidataInstanceOf` (Q15632617 = fictional human, Q4663903 = fictional video game character) to constrain the result set.

**Correct approach — two changes:**

**Change 1 — categories.ts:** Remove `wikidataGender` from `fictional-women` and `fictional-men` config entries. The instanceOf constraint (`['Q15632617', 'Q4663903']`) is specific enough to avoid returning non-fictional characters. Leave `wikidataGender` on `women` and `men` (real people categories where gender is reliable on Wikidata).

The fictional-women entry should become:

```typescript
{

  // wikidataGender intentionally omitted: many fictional chars lack P21 on Wikidata
  wikidataInstanceOf: ['Q15632617', 'Q4663903'],
},
```

Apply the same removal to `fictional-men` (remove `wikidataGender: 'Q6581097'`, keep `wikidataInstanceOf`).

**Change 2 — wikidata.ts:** Apply the conditional gender filter fix shown above. The filter uses `!category.wikidataGender` to skip when not set.

Both files already have the correct `wikidataInstanceOf: string | string[]` type and array handling — do NOT change those parts, only apply the two targeted changes above.
