<task type="auto">
  <name>Task 1: Run famous-asians build script and verify output quality</name>
  <files>src/data/allowlist-famous-asians.json</files>
  <action>
Run the existing build script for the famous-asians category. No API key required — it queries Wikidata's public SPARQL endpoint.

```bash
node scripts/build-allowlist.js --category famous-asians
```

The script will:
1. Send a SPARQL query to `https://query.wikidata.org/sparql` fetching humans (P31=Q5) with citizenship in 20 Asian countries (P27), filtered to >= 20 Wikipedia sitelinks, ordered by sitelink count descending, LIMIT 3000
2. Deduplicate by name
3. Write output to `src/data/allowlist-famous-asians.json`

Expected output in console: `Wrote N famous Asians to ...` where N should be 500-3000.

**If the SPARQL query times out** (Wikidata's public endpoint has a 60s timeout for complex queries), the script will throw. In that case, run the query in two halves by modifying the SPARQL in the script temporarily: first add `FILTER (?sitelinks >= 50)` to get a smaller high-confidence set, run it, then check if the count is sufficient (>= 300 entries is acceptable for a playable game).

**After the script runs**, verify the output:
```bash
node -e "const d = require('./src/data/allowlist-famous-asians.json'); console.log('entries:', d.length, '| first 3:', d.slice(0,3).map(e=>e.name).join(', '))"
```

Expected: `entries: 500+` with recognizable names like "Jackie Chan", "Bruce Lee", "Yoko Ono", "Narendra Modi".

If the count is below 200, lower the sitelinks threshold in the SPARQL from 20 to 10 and re-run.

**Check for obvious data quality issues:**
- Names starting with `Q` (bare QIDs leaked through) — these should be 0 (the script already filters them: `.filter(r => !r.personLabel.value.startsWith('Q'))`)
- Duplicate names — should be 0 (the script deduplicates)
- Non-person entries — spot-check 10 random entries

**Commit the populated file:**
```bash
git add src/data/allowlist-famous-asians.json
git commit -m "data(famous-asians): populate allowlist via Wikidata SPARQL (N entries)"
```
Replace `N` with the actual count.
