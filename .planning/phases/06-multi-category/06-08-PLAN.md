Run the existing men builder:

```bash
node scripts/build-allowlist.js --category men
```

**What the script does:**
1. Fetches the GitHub Twitch top-streamers CSV (CC0, no auth needed): `https://raw.githubusercontent.com/phelpsbp/Twitch-Streamer-Analysis/main/twitchdata-update.csv`
2. If `TWITCH_CLIENT_ID` + `TWITCH_CLIENT_SECRET` env vars are set: also fetches live Twitch streams
3. If `YOUTUBE_API_KEY` is set: also fetches YouTube creators
4. If `ANTHROPIC_API_KEY` is set: sends batches to Claude Haiku for male gender classification
5. Runs Wikidata verification on LLM-confirmed names
6. Writes output to `src/data/allowlist-men.json`

**Without any API keys:** The script will still run using only the GitHub CSV dataset, skip LLM classification (all candidates pass through unfiltered), and skip Wikidata verification. This produces a larger but less-curated list. That is acceptable — the Wikidata game-time verification acts as a second filter for real-name celebrities anyway.
